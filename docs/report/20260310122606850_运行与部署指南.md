# 运行与部署指南

## 适用范围
- 桌面端首次使用
- Service 版本独立运行
- Docker 部署
- macOS 首次放行

## 快速开始
1. 启动桌面端，点击“启动服务”。
2. 进入“账号管理”，添加账号并完成授权。
3. 如回调失败，粘贴回调链接手动完成解析。
4. 刷新用量并确认账号状态。
5. 如需管理本机 Codex OAuth 登录态，进入侧边栏“本地 Codex”页面扫描并导入当前本机登录。
6. 如需让 IDE / CLI 使用账号池和请求日志，请创建平台 Key，并让客户端改走 CodexManager 本地监听地址。

## 本地 Codex 工作台

桌面端提供独立的“本地 Codex”页面，负责处理本机 `~/.codex` 登录态。

支持能力：
- 扫描 `auth.json`、`config.toml`、`state_5.sqlite`
- 导入当前本机 Codex OAuth 登录态到账号池
- 在多个已保存的 Codex OAuth 登录记录之间一键切换，并写回本机 `auth.json`
- 展示 `config.toml` 中的已信任项目范围

说明：
- 这里切换的是“本机 Codex 当前登录态”，不会自动替代网关里的“手动优先账号”策略。
- 同一 `workspace_id` 下允许保留多条不同 token 记录，用于多个 business / workspace 登录态切换。
- 如果某个 business 从未导入进 CodexManager，就不会出现在切换列表里。

## 账号导入与导出
- `批量导入`：选择多个 `.json/.txt` 文件后统一导入。
- `按文件夹导入`：仅桌面端可用；选择目录后递归扫描其中 `.json` 文件并批量导入，空文件会自动跳过。
- `导出用户`：选择目录后按“一个账号一个 JSON 文件”导出，便于备份与迁移。

## Service 版本
1. 下载 Release 中的 `CodexManager-service-<platform>-<arch>.zip` 并解压。
2. 推荐启动 `codexmanager-start`，一个进程拉起 service + web，可直接在控制台 `Ctrl+C` 关闭。
3. 也可以只启动 `codexmanager-web`，它会自动拉起同目录的 `codexmanager-service` 并打开浏览器。
4. 或者先启动 `codexmanager-service`，再启动 `codexmanager-web`。
5. 默认地址：service `localhost:48760`，Web UI `http://localhost:48761/`。
6. 关闭：访问 `http://localhost:48761/__quit`；若 web 自动拉起过 service，会尝试一并关闭。

## 请求日志说明

- “请求日志”只记录经过 CodexManager 本地网关的请求。
- 如果你是在官方 Codex CLI、本机 `~/.codex` 登录态、ChatGPT / Codex 网页里直接发请求，这些流量不会出现在请求日志页面。
- 如果你希望看到日志，需要让客户端把 `Base URL` 指到 CodexManager 的本地监听地址，并使用平台 Key。

## Docker 部署

### 方式 1：docker compose
```bash
docker compose -f docker/docker-compose.yml up --build
```

浏览器打开：`http://localhost:48761/`

### 方式 2：分别构建和运行
```bash
# service
docker build -f docker/Dockerfile.service -t codexmanager-service .
docker run --rm -p 48760:48760 -v codexmanager-data:/data \
  -e CODEXMANAGER_RPC_TOKEN=replace_with_your_token \
  codexmanager-service

# web（需要能访问到 service）
docker build -f docker/Dockerfile.web -t codexmanager-web .
docker run --rm -p 48761:48761 \
  -v codexmanager-data:/data \
  -e CODEXMANAGER_WEB_NO_SPAWN_SERVICE=1 \
  -e CODEXMANAGER_SERVICE_ADDR=host.docker.internal:48760 \
  -e CODEXMANAGER_RPC_TOKEN=replace_with_your_token \
  codexmanager-web
```

- 注意：若希望 Web 密码、设置项、缓存模型列表等状态与 service 保持一致，`codexmanager-web` 必须和 `codexmanager-service` 共享同一份 `/data` 卷。

## macOS 首次启动
- 当前 macOS Release 产物未使用 Apple Developer 账号完成公证，因此首次从浏览器下载后，Gatekeeper 可能提示“已损坏”或拒绝打开。
- Release 中的 macOS `dmg` 已内置 `Open CodexManager.command` 与 `README-macOS-first-launch.txt`。
- 推荐先把 `CodexManager.app` 拖到“应用程序”，再双击该脚本完成首次放行。
- 也可以直接执行：

```bash
xattr -dr com.apple.quarantine /Applications/CodexManager.app
```

- 如果仍被拦截，再对 `CodexManager.app` 执行一次“右键 -> 打开”。

## Windows 打包说明

- `pnpm -C apps run tauri:build:windows` 只能在 Windows 主机上执行，或放到 GitHub Actions 的 `windows-latest` runner 里执行。
- 在 Linux / macOS 上运行这个命令不会产出 Windows 安装包；当前脚本会直接给出明确提示并退出。
- Windows 下本机 Codex 目录会按 `HOME -> USERPROFILE -> HOMEDRIVE + HOMEPATH` 顺序探测，避免只依赖 `HOME`。
